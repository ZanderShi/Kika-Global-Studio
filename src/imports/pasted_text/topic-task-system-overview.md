项目名称：全球设计师平台 Topic 任务系统 1.0

功能背景：
全球设计师平台当前允许设计师自行创建资源。新功能引入「Topic 任务」机制，由运营/管理人员创建任务并分配给设计师。设计师不再主动创建资源，而是从 Home / My Work 中领取和制作被分配的任务。系统需要支持任务创建、批量创建、设计师分配、时间管理、超时提醒、标签同步、资源名锁定、任务快照展示，以及和运营平台标签系统的数据打通。

核心目标：
1. 让运营可以创建 Topic 任务，并分配给指定设计师。
2. 让设计师只能在被分配的任务中制作资源，不再展示 Create 入口。
3. 让平台可以管理设计师时间占用、任务排期和超时状态。
4. 让 Topic 中填写的资源信息贯穿资源制作流程，设计师不可随意修改资源名。
5. 让全球设计师平台选择的标签单向同步到运营平台。

用户角色：
- 运营 / 管理员：可以创建 Topic、编辑 Topic、分配任务、撤回任务、重新分配任务、管理设计师时间、维护运营和设计师用户。
- 设计师：只能查看和制作分配给自己的任务；不能主动创建任务；不能修改 Topic 快照和资源名。
- 系统：负责状态计算、超时判断、时间释放、标签同步、逾期任务回收。

权限规则：
- Topic 首页需要权限控制，权限逻辑同 Manage。
- Topic 上线后，设计师不可见 Create 入口。
- Create 功能本身保留，不删除，只隐藏入口，后续资源品类仍从此能力扩展。
- 已同步资源不允许编辑 Topic，编辑按钮置灰，点击提示 toast：“资源已同步，不支持修改”。

一、Topic 首页

页面用途：
展示所有 Topic 任务，支持筛选、状态统计、查看任务信息、分配、编辑、删除、撤回、重新分配。

顶部状态统计：
- 总数：待分配 + 进行中 + 已完成。
- 待分配：创建后尚未指派设计师。
- 进行中：未开始 + 平台现有的其他处理中状态。
- 已完成：审核通过的资源。

状态定义：
- 待分配：Topic 创建后，未指派设计师。
- 未开始：已指派设计师，但仍在设计师 My Work 中，等同原草稿状态。
- 其他状态：沿用平台当前资源状态，如实展示。
- 已完成：审核通过。
- 超时：当前北京时间超过任务结束时间，且任务未完成。

筛选项：
- 资源类型：Themepack、Control Center、Supertheme，支持多选。
- 归属应用：Themepack iOS、Themepack Android、ithemes、Cooltheme、Themely，支持多选，选项写死。
- 跟进运营：平台维护数据，支持多选。
- 设计师：平台维护数据，支持多选。
- 状态：待分配、进行中、已完成，支持多选。
- 是否延期：延期、正常，支持多选。

列表字段：
- 名字 / 描述：创建 Topic 时录入。固定长度展示，超出省略号；鼠标悬浮展示完整内容。
- Topic 图片：展示 1 到 3 张缩略图；点击可放大；放大后支持查看当前 Topic 的上一张 / 下一张。
- 任务时间：分配后展示。展示任务起止时间、剩余时间；超时用红色展示。
- 跟进运营：展示 Topic 创建时选择的运营。
- 设计师：展示当前分配设计师。
- 状态：展示当前 Topic 状态。
- 操作按钮：分配、编辑、删除、撤回、重新分配。

操作规则：
- 分配：打开分配弹卡。
- 编辑：未同步资源可编辑，编辑后 Topic 数据同步变化；已同步资源不可编辑。
- 删除：删除 Topic。
- 撤回：将任务从设计师身上撤回，状态变为待分配。
- 重新分配：等于先撤回再分配。
- 逾期 2 个月未完成的 Topic 自动回收到待分配状态。

时间展示规则：
- 所有时间以中国北京时间为统一标准。
- 0:00 到 14:00 视为上午。
- 14:00 到 24:00 视为下午。
- 最小时间粒度为 0.5 天。

二、创建 Topic 页面

页面用途：
运营创建单个 Topic 或批量创建 Topic，填写资源基础信息、图片、描述和标签。

字段要求：
- 资源类型：单选，选项为 Themepack、Control Center、Supertheme，必填。
- 归属 App：多选，选项为 Themepack iOS、Themepack Android、ithemes、Cooltheme、Themely。
- 跟进运营：单选，数据来源于运营管理，必填。
- 资源命名：使用平台统一命名规则；资源内命名唯一；后续设计师制作流程中不可修改。
- 相关图片：支持拖拽上传、粘贴上传；支持静态图和动态图；数量不限制；格式不限制。
- 详细描述：自定义输入，字数不限制。
- 关联标签：展示已选标签区域，需要去重。

批量创建：
- 支持 Excel 上传。
- 批量创建表单只需要资源类型和跟进运营。
- 资源名字可以先不填写。
- 批量创建入口需要在创建 Topic 页面中体现。

边界情况：
- 如果资源类型、跟进运营、资源命名等必填项缺失，点击分配时 toast 提示“缺少 xxx，无法进行分配”。
- 上线时未完成的历史资源，继续按老流程完成，不强制迁移到新 Topic 流程。

三、标签选择与标签弹层

功能背景：
全球设计师平台和运营平台使用同一套标签数据系统。两侧都能维护标签，但同步方向为单向。

标签层级：
- 一级标签：IP、二创、普通、版权。
- 默认选中：普通。
- 选中一级后展示二级标签。
- 二级支持多选。
- 二级下有三级时展示三级标签，三级支持多选。
- 展示二级下三级数量；数量为 0 时不展示。
- IP 和二创的二级、三级使用同一套数据。
- 一级、二级、三级都作为标签赋予给资源。

已选标签区域：
- 单独区域展示当前已选标签。
- 标签必须去重。
- 多选二级、三级时要清晰展示层级关系。

搜索规则：
- 支持搜索标签。
- 如果搜索同时命中二级和三级，需要让用户选择。
- 用户选中搜索结果后，自动展开对应层级数据。

创建 / 编辑标签：
- 支持创建和编辑标签。
- 同级别标签不允许重复。
- 例如：二级不能出现两个“狗”，即使它们位于不同一级下；三级同理。
- 编辑作用于唯一标签数据库。
- 已经被资源使用过的标签，不因编辑发生数据增删改异常。

标签同步：
- 全球设计师平台选中的标签，在资源同步时带到运营平台。
- 同步为单向注入：全球设计师平台 -> 运营平台。
- 运营平台选择或修改的标签，不反向注入全球设计师平台。

四、分配任务弹卡

打开时机：
点击 Topic 首页的“分配”按钮后打开。

打开前校验：
- 检查资源类型、跟进运营、资源命名是否已填写。
- 如果不完整，toast 提示缺少字段，阻止分配。

设计师选择：
- 展示设计师列表，单选。
- 增加“最近使用”前置逻辑。
- 最近使用以用户为单位，不同运营有不同数据。
- 支持按设计师组筛选。
- 记录用户上次选择的设计师组。

时间选择：
- 最小粒度 0.5 天，即上午 / 下午。
- 统一按北京时间计算，不因使用地变化。
- 选择开始时间和结束时间，中间时间自动带上。
- 单个任务最长 30 天。
- 所有时间平等，不设置节假日。

时间与设计师互相筛选：
- 如果先选设计师，时间区域展示该设计师已有占用。
- 如果先选时间，则根据时间筛选可选设计师，不可选设计师置灰。
- 一期同一时间段最多只能有 1 个任务。
- 后续可能扩展为同一时间段最多 2 个任务。

任务查看：
- 鼠标悬浮占用时间块，展示任务详情。
- 详情用左侧弹卡承接，仅展示，不提供交互。
- 运营可以在当前页面修改任务时间范围。
- 修改方式为拖动时间条。

特殊处理：
- 如果任务提前完成，剩余未使用时间需要释放出来。
- 释放后该时间段可重新分配给其他任务。

五、设计师时间管理

页面用途：
让运营查看所有设计师的任务排期和时间占用，并可自定义占用时间。

设计师首页：
- 展示所有设计师从过去到未来 30 天的时间安排。
- 当前页面第一列为当前日期。
- 向右滑动查看未来时间。
- 向左滑动查看过去时间。
- 过去时间展示范围需考虑技术成本：优先全量记录；如成本较高，至少展示半年。
- 需要处理历史未完成 Topic。
- 逾期 2 个月未完成的 Topic 自动回收到待分配。

自定义占用：
- 可在设计师管理首页自定义占用设计师时间。
- 自定义占用需要输入标题和时间。
- 自定义占用支持删除。

颜色规则：
- 浅灰：已完成。
- 红色：超时任务。
- 黄色：自定义占用。
- 深灰：未开始。
- 蓝色：进行中。

悬浮弹卡：
- 鼠标悬浮时间块，展示当前任务状态弹卡。
- 分配任务类型 Themepack / Control Center / Supertheme 属于任务占用。
- 任务占用弹卡展示：资源名字、资源类型、跟进运营、Topic 分配人、任务时间。
- 自定义占用弹卡展示：标题、时间，并支持在设计师管理首页删除。

修改时间：
- 支持通过拖动时间条修改任务时间。
- 点击某个设计师，进入该设计师个人时间安排页。
- 个人时间安排页也支持修改时间。

六、运营和设计师维护

页面用途：
维护平台中哪些用户是运营、哪些用户是设计师，以及设计师所属分组。

功能要求：
- 展示平台全部用户。
- 通过勾选 / 取消勾选设置用户角色。
- 点击保存后立即生效。
- 设计师管理增加组归属。
- 一期默认写死 10 个设计师组。
- 如果后续组数量持续增加，再升级为组管理功能。

七、Home / My Work 设计师端

功能背景：
原草稿箱变为 My Work。设计师不能主动创建资源，只能制作分配给自己的任务。

入口规则：
- Create 入口隐藏。
- Home 仍然展示给所有注册用户。
- My Work 展示设计师被分配的任务。
- 原草稿未提交前仍可保存草稿，保存后仍在 My Work 中，状态不变。

空状态：
- 去掉原缺省图中的 Create 按钮。
- 空状态文案改为：
  “Please contact your work partner to assign tasks to you.”

任务来源：
- 运营创建 Topic 时选择资源类型。
- Topic 分配给哪个设计师，就出现在该设计师对应资源品类的任务列表中。

My Work 任务卡片：
- 删除原来的设计师名字展示。
- 删除原来的状态展示，因为 Tab 已能代表状态。
- 增加任务时间，展示由 Topic 分配时间换算的总工时。
- 增加跟进运营，来源于 Topic 创建。
- 增加分发时间。
- 保留现有消息能力。
- 因已有明确跟进运营，消息最好直接对话对应运营。
- 消息能力最好对接钉钉对话窗口。
- 卡片左下角展示 Topic 入口，点击后展示 Topic 快照。
- 点击“去制作”后，根据资源类型进入对应制作流程。
- 设计师侧制作流程内，Topic 均展示为快照，不可修改。

其他状态任务卡片：
- 删除设计师名字展示。
- 删除原状态展示。
- 增加任务时间，来源于 Topic 分配。
- 增加跟进运营，来源于 Topic 创建。
- 增加提交时间。
- 保留消息能力，最好对接跟进运营的钉钉对话。
- 左下角展示 Topic 快照。
- “去制作”根据资源类型进入对应流程。

八、Home 全状态超时与排序

适用范围：
除 Approved 状态外，所有状态都需要展示任务和超时提示。

顶部资源类型 Tab：
- 有任务时持续展示蓝色点。
- 原有红点逻辑保持。
- 有超时任务时持续展示红点。
- 蓝点和红点同时存在时，只展示一个提示点，需要优先表达超时风险。

My Work 状态位置：
- 有任务时展示蓝色提示，并展示任务数量。
- 有超时任务时展示红色提示，并展示超时任务数量。

卡片超时展示：
- 超时任务顶部展示 timeout 标识。
- 超时任务卡片整体变红。
- 任务时间区域整体变红。

排序规则：
- 除 Approved 状态外，超时任务排在最前。
- 正常任务排在后面。
- 正常任务按结束时间从近到远排序。
- 时间相同不需要三级排序，依次展示即可。

Approved 状态：
- 已完成任务不再展示超时和任务提醒。
- 卡片增加实际完成时间区间。
- 列表排序可保持当前线上逻辑，也可按完成时间从近到远排序。

九、资源名锁定

功能背景：
资源名称由 Topic 创建阶段确定，设计师制作过程中不得修改。

规则：
- 所有类型的 Resource Name 在设计师制作全流程中锁死，不可修改。
- Themepack：首页资源名字锁死，取 Topic 创建时填写的名称。
- Control Center：资源名字不可编辑，取 Topic 创建时填写的名称；Rename 保持原状。
- Supertheme：Rename 中的资源名字锁死，取 Topic 创建时填写的名称。

边界情况：
- 如果 Topic 上线时存在未完成的老资源，仍按老流程完成。
- 新 Topic 创建的资源必须遵循资源名锁定规则。
- 已同步资源不可再编辑 Topic。

十、Manage 页面

功能要求：
- 在 Manage 中为每个资源增加 Topic 入口。
- 只要资源未同步，就可以编辑 Topic。
- 已同步资源不可以编辑 Topic。
- Manage 全流程中，除已同步状态外，Topic 都能修改。

十一、工时管理表

一期 / 二期边界：
- 当前文档提到工时管理表为二期内容。
- 二期需要记录实际分配时间和实际完成时间。
- 二期需要计算工时差值。
- 可参考附件：工时管理模板.xlsx。

十二、任务提醒

提醒类型：
- 新任务提醒。
- 即将超时提醒。
- 已超时提醒。

实施边界：
- 邮件提醒为二期。
- 一期 UI 需要至少体现任务状态、超时标识、红点 / 蓝点提示、timeout 标签。
